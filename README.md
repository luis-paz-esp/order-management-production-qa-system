# Order Management & Production Planning QA System

## Project Overview

This QA portfolio project focuses on validating a real order management workflow used to capture, organize, and process weekly customer orders.

The system includes order intake, conditional form logic, customer data validation, duplicate record detection, product selection, payment status, production planning, weekly requisition generation, and AI-assisted process automation.

The main goal is to ensure that order data is complete, consistent, and reliable enough to support downstream operational tasks such as production planning and requisition generation.

## Main Objective

To validate that the order management workflow captures accurate customer and order information, prevents incomplete submissions, detects duplicate or inconsistent records, and supports the generation of reliable production and requisition outputs.

## Business Problem

Manual order collection can lead to incomplete records, duplicate customer names, missing preferences, unclear payment status, and errors during production planning.

This project documents how QA practices can be applied to validate and improve the reliability of a real order management workflow.

## Expected Impact

- Reduce incomplete order records
- Improve data consistency
- Detect duplicate customer records
- Improve production planning accuracy
- Reduce manual consolidation errors
- Support weekly requisition generation
- Create a foundation for future automation
